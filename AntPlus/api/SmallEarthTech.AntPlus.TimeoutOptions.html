Remarks

The application host builder retrieves configuration options from various sources such as the command line or appsettings.json. If the configuration can't find TimeoutOptions, the collection will default to MissedMessages. If both Timeoutand MissedMessagesare defined, Timeout is ignored and MissedMessages is used. Prefer MissedMessages as this will scale the ANT device timeout based on the specific data broadcast rate of an ANT device. Asset trackers broadcast 16 times a second and Geocaches broadcast once every 2 seconds. Other devices typically broadcast 4 times a second.

For example, your appsettings.json file would have this entry - 

{
  "TimeoutOptions": {
    "MissedMessages": 8,
    "Timeout": null
  }
}
The command line could define the timeout "--TimeoutOptions:MissedMessages=8".

Timeouts can be disabled by setting Timeout to -1 and ignoring or setting MissedMessages to null.
