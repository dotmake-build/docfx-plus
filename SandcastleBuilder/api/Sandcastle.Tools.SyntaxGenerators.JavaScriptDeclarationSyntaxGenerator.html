Remarks

This version differs from the ScriptSharpDeclarationSyntaxGeneratorin that it looks for a <scriptSharp />element in the <api>node and, if found, only then will it apply the casing rules to the member name. If not present, no casing rules are applied to the member names thus it is suitable for use with regular JavaScript such as that used in AjaxDoc projects.

In order to use this script generator with Script# content, the Sandcastle reflection data file must first be transformed using the Script# Reflection File Fixer plug-in so that the necessary changes are made to it.
